# Exam Flow Plan — 2026-05-15

## Goal
Support two exam entry points:
1. **Template-based**: User starts exam from a template → generate a new exam instance on the fly
2. **Instance-based**: User starts exam from a specific exam instance → use that exact instance

Both flows converge on the same exam-doing page at `/do`.

---

## Backend

### New endpoint: `POST /exams/instances/{instance_id}/attempts`
- `src/routes/exam_route.py` → `create_exam_attempt_from_instance()`
- `src/services/exam_attempt_service.py` → `start_attempt_from_instance(instance_id, user)`
- Validates instance is not REJECTED, ensures it has eligible questions
- Creates attempt record and returns same `ExamAttemptResponse` shape

### Existing endpoint (unchanged): `POST /exams/templates/{template_id}/attempts`
- `use_existing_instance=false` → generate new instance via LLM
- `use_existing_instance=true` → pick random existing instance

---

## Frontend

### Route: `/(auth)/(general)/do/index.tsx`
Search params (validated with zod):
```
exam_identity_type: 'template' | 'instance'
exam_identity_id: string
use_existing_instance?: boolean   (only meaningful for template)
```
Renders `<UserDoingExamPage examIdentityType={...} examIdentityId={...} />`

### Page: `UserDoingExamPage`
Props changed from `{ templateId, useExistingInstance }` to:
```ts
{
  examIdentityType: 'template' | 'instance'
  examIdentityId: string
  useExistingInstance?: boolean
}
```
- Builds `examIdentityKey = "{type}:{id}"` for stale-state detection
- For `template`: calls `useCreateExamAttempt(examIdentityId)`
- For `instance`: calls `useCreateExamAttemptFromInstance(examIdentityId)`
- Rest of flow (token, questions, auto-save, submit, result) unchanged

### Store: `exam-store.ts`
Added `examIdentityKey: string | null` field to detect stale sessions across both identity types (replaces relying solely on `templateId`).

### Navigation changes
- `UserExamTemplateDetailPage` "Start Exam" → `/do?exam_identity_type=template&exam_identity_id={templateId}`
- `UserExamInstanceDetailPage` "Start Exam" → `/do?exam_identity_type=instance&exam_identity_id={instanceId}`
